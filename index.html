类：
程序的数据和功能被组织为逻辑上相关的数据项和函数的封装集合，被称为类，类是一个能储存并执行代码的数据结构。他包含 数据成员 和 函数成员。

数据成员：字段，常量。表示类或类的实例相关的数据（有什么）
函数成员：方法，属性，运算符，索引器，事件。表示类的实例能执行的操作（能干什么）

Java笔记——抽象类

发表于 2018-05-28 | 分类于 JAVA

关键字：abstract
作用：限制所有该抽象类的子类都有该抽象类的操作。

注意：
（1）抽象类不能创建对象，必须派生出子类由子类创建对象。
（2）抽象方法只允许声明不允许实现，即没有方法体。
（3）抽象类的子类必须重写父类中的所有抽象方法。
（4）抽象类中可以没有抽象方法，抽象方法必须放在抽象类中。

Java笔记——Object类

发表于 2018-05-28 | 分类于 JAVA

Object类： java中所有类的父类。

可以使用Object类型的变量引用任何类型的对象，但若想对其中的值进行操作还需要搞清对象的原始类型，并进行相应的类型转换。

Object类中的方法

equals方法

equals方法用来检测一个对象是否等于另一个对象。

“==”比较的是内存地址，所以含有复合型数据的实例变量再调用equals时应根据需求对equals()进行重写。

注意：对简单数据类型进行封装的Boolean、Byte、Char、Short、Integer、Long、Float、Double以及String，Date，File类把equals方法重写为判断 值 是否相等。

toString方法

toString方法用来返回对象的字符串表示
在直接输出一个对象时，该对象自动调用toString方法，有时这个方法需要根据要求重写。

Java笔记——接口

发表于 2018-05-25 | 分类于 JAVA

接口的概述

1.接口中只能定义常量和抽象方法。
2.接口中的方法默认为public，abstract。

接口的实现

声明

在类的声明中用implements关键字来表示在类体中可以使用接口中的常量，而且必须实现（重写）接口中的所有方法。若使用多个接口则需以逗号隔开接口名。

回调

其他

1.接口可以作为引用类型来使用
2.任何接口类型的实例都存储在该接口类型的变量中
3.接口中的方法默认为public，所以在重写方法时应限定为public

Java笔记——继承,多态

发表于 2018-05-24 | 分类于 JAVA

继承

（1）其中Subclass为子类，extends为继承关键字，Superclass为父类。
（2）在继承中，子类可以看作父类的一个子集。
（3）Java中只能单一继承，不支持多重继承，但可以多层继承。

父类成员的隐藏

子类和父类中出现同名的变量和方法时，调用时默认使用子类中的成员（就近）。若想调用父类中的成员，则需要引入super关键字。

方法重写

子类中出现与父类同名的方法

原因：方法操作相同，但父类中的该方法已经满足不了子类中对该操作的要求。

原则：
（1）方法名，返回类型，参数列表一摸一样。
（2）重写后的方法不能有更严格的保护权限。
（3）重写后的方法不能产生更多异常。

问题：
子类构造方法会有先调用父类中无参的构造方法。
解决办法:
（1）在父类中重载一个无参的构造方法。
（2）在子类构造方法的第一句添加super.([参数列表])手动调用父类中无参的构造方法。

注意：super处取什么实参取决于父类构造方法中的型参。

多态

多态性：由于方法重载和覆盖引起的方法同名不同功能现象。

分类：静态多态性和动态多态性

静态(编译时)多态性：方法重载，如println方法

动态(运行时)多态性：多个子类继承同一个父类，并各自重写父类的方法，那么运行时可以完成不同的功能。

发表于 2018-05-21 | 分类于 JAVA

问题：

Java类的名称是该类的唯一标识，编译后产生的字节代码文件名和类名相同。因此在同一个文件夹中不能存在两个同名的类。

解决方法：

将字节代码文件存放在不同的子文件夹中，并把相对路径和类名混合在一起，做为类的完整名。

包

1.定义：

放置多个字节代码文件的文件夹路径名。它是用于管理类名的特殊手段。

2.package语句：

用package语句定义类所在的包。格式为：

代码高亮

注意：
（1）如果用package语句，它必须是源程序中的第一条语句；
（2）含有package语句的源程序中所有的类都属于该包；
（3）不能指定到文件上，如”package p1.SameName”只能说明后续的类是p1.SameName包中的类；
（4）编译包含该语句的Java程序后，需要将编译生成的所有的class文件拷贝到当前目录下的pkg1\pkg2\pkg3…目录内，否则不能正确访问这些类；
（5）包名与源程序的编译没有任何关系，只在运行类和实例化类时起作用；
（6）包在命名时应该全小写；
（7）包的名字应该与类的名字相异。

1.源代码第一行加入package语句。
2.编译的时候使用 -d 选项（如： javac -d . A.java）
3.运行包中的类时应使用完整类名。
