父类成员的隐藏

子类和父类中出现同名的变量和方法时，调用时默认使用子类中的成员（就近）。若想调用父类中的成员，则需要引入super关键字。

方法重写

子类中出现与父类同名的方法

原因：方法操作相同，但父类中的该方法已经满足不了子类中对该操作的要求。

原则：
（1）方法名，返回类型，参数列表一摸一样。
（2）重写后的方法不能有更严格的保护权限。
（3）重写后的方法不能产生更多异常。

问题：
子类构造方法会有先调用父类中无参的构造方法。
解决办法:
（1）在父类中重载一个无参的构造方法。
（2）在子类构造方法的第一句添加super.([参数列表])手动调用父类中无参的构造方法。

注意：super处取什么实参取决于父类构造方法中的型参。

多态

多态性：由于方法重载和覆盖引起的方法同名不同功能现象。

分类：静态多态性和动态多态性

静态(编译时)多态性：方法重载，如println方法

动态(运行时)多态性：多个子类继承同一个父类，并各自重写父类的方法，那么运行时可以完成不同的功能。

Java笔记——包

发表于 2018-05-21 | 分类于 JAVA

问题：

Java类的名称是该类的唯一标识，编译后产生的字节代码文件名和类名相同。因此在同一个文件夹中不能存在两个同名的类。

解决方法：

将字节代码文件存放在不同的子文件夹中，并把相对路径和类名混合在一起，做为类的完整名。

包

1.定义：

放置多个字节代码文件的文件夹路径名。它是用于管理类名的特殊手段。

2.package语句：

用package语句定义类所在的包。格式为：

代码高亮

注意：
（1）如果用package语句，它必须是源程序中的第一条语句；
（2）含有package语句的源程序中所有的类都属于该包；
（3）不能指定到文件上，如”package p1.SameName”只能说明后续的类是p1.SameName包中的类；
（4）编译包含该语句的Java程序后，需要将编译生成的所有的class文件拷贝到当前目录下的pkg1\pkg2\pkg3…目录内，否则不能正确访问这些类；
（5）包名与源程序的编译没有任何关系，只在运行类和实例化类时起作用。

Java p1.p2.SameName

（3）import只能导入包中的类，不能导入包中的包。

把一个类放到包中再执行的过程

1.源代码第一行加入package语句。
2.编译的时候使用 -d 选项（如： javac -d . A.java）
3.运行包中的类时应使用完整类名。
